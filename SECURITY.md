# Security Policy

## Supported versions

The plugin is maintained as a single line of development. Security fixes are released against the latest version, so always run the most recent release.

| Version | Supported |
|---|---|
| 1.0.x | Yes |
| < 1.0 | No |

## Reporting a vulnerability

Please report security vulnerabilities privately, not through a public issue or discussion.

The preferred channel is GitHub's private vulnerability reporting: open the **Security** tab of [the repository](https://github.com/Kntnt/kntnt-autolink/security/advisories) and choose **Report a vulnerability**. If you cannot use that, email thomas@kntnt.com instead.

Please include enough detail to reproduce the issue: the affected version, the conditions under which it occurs, and a proof of concept where possible.

## What to expect

You can expect an acknowledgement of your report within a few working days. Once the issue is confirmed, a fix is prepared and released, and the report is credited unless you prefer to remain anonymous. Please give a reasonable amount of time for a fix to ship before disclosing the issue publicly.
